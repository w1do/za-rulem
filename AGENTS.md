
# План проекта: ZA-RULEM

Этот файл содержит описание основной идеи проекта, спецификацию , правила по разработке и SEO

## 💡 Идея проекта

**Название** — это сайт `Автопомощи на дороге в Тюмени`

## Главный файл
Открывай всегда главный файл, когда начинаешь работу с проектом `.junie/skills/SKILL.md`

## SEO, написание контента
Используй навык `.junie/skills/seo-write` этот навык описывает все правила правильного написания контента для страницы

## Написание текстов от лица разработчика
Никаких, вы, вас, компании, студии, все тексты пишутся от лица разработчика в единственном лице, меня, я  
Пример: я разработал систему, я реализовал крутой кейс, меня приняли в команду

## Написание посадочных страниц
Когда требуется охватить один запрос разными темами используй навык `.junie/skills/cluster-pilar-page`, есть главный запрос например `Создание чат ботов`, и идет уже расширение страниц по кластеру `какие боты`, `для чего используются` и.т.д, подключи навык `.junie/skills/seo-write` для правильного написания SEO страниц

## Вёрстка и исходный рефер!енс
Вёрстка шаблона лежит по пути `source/`, требуеться создавать страницы в точности как там (путём переноса html), меняются только тексты на `русском языке`, от себя не добавляй новые стили, блоки, и другие визуальные моменты, и не фонтазируй, запросы для описания страницы берутся из `seo/` под запрос, и используется навык по заполнению страницы и структуры по навыку `.junie/skills/seo-write`  
Для форм, обратных заявок, обязательно функционал выносится в отдельные хуки на react

## Копия страницы для посадочных страниц
Следуй всем правилам из файла по созданию структуры страницы для посадочной под запрос `.junie/skills/landing-copy`

### Основные задачи по архитектуре:
- Соблюдать простую не сложную архитектуру FSD, выносить компоненты в `ui`, `shared`, `forms`
- Наполнить статическим контентом через `content/` и коллекции .md
- Подготовить сайт для лучшей индексации сайта в поисковых системах собранный из ключевых слов `seo/`
- Сделать максимально быстрым простым в обслуживании
- Удобные места для переиспользования контента mocks/ и компонентный подход где каждый компонент дробиться на множество других
- Наполнить сайт статическим контентом по сео правилам

### Технологический стек:
- **Framework**: Astro (SSG/SSR)
- **Styling**: Tailwind CSS
- **UI**: Shadcn UI + Lucide Icons
- **Content**: Markdown (Content Collections)
- **SEO**: `.junie/skills/seo-write/SKILL.md` — Правила написания оптимизированного контента
- **Архитекстура**: `.junie/skills/architecture/SKILL.md` — архитектура проекта

### MCP написания контента
Каждый раз когда нужно написать контент, (`пост`, `статью`, `страницу`, `посадочный материал`, `сео наполнение`) по тематике сайта обращайся к MCP `rag-content-botsync-w1do` с запросом и id шаблона для написания контента по обученому RAG  
Пример: напиши статью на тему `где применяются ии боты` в кластере услуг, используя шаблон под id 11
