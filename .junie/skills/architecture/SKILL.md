### Архитектура проекта

Проект строится на базе фреймворка **Astro** с использованием статической генерации (SSG) и частичного SSR для динамических форм.

#### 1. Структура директорий

- `src/content/` — Основное хранилище контента (Content Collections). Каждая подпапка — отдельная коллекция (blog, reports, locations, events, market, faq).
- `src/components/` — UI-компоненты (Shadcn + кастомные).
  - `ui/` — Базовые компоненты Shadcn.
  - `shared/` — Общие компоненты (Header, Footer, Cards).
  - `home/` — Специфичные для главной страницы блоки.
- `src/layouts/` — Шаблоны страниц (основной Layout).
- `src/pages/` — Роутинг страниц.
- `src/schemas/` — Zod-схемы для валидации контента.

#### 2. Content Collections

Использование `src/content` является приоритетным методом работы с данными:

- **Blog**: Статьи о рыбалке и кемпинге.
- **Reports**: Отчеты, привязанные к `locationId`.
- **Locations**: Карточки водоемов и мест.
- **Events**: Информация о событиях.
- **Market**: Объявления барахолки.
- **FAQ**: Вопросы и ответы по категориям.

#### 3. Технологический стек

- **Framework**: Astro 4.x
- **Language**: TypeScript (строгая типизация)
- **Styling**: Tailwind CSS
- **UI Components**: Shadcn UI (на базе Radix UI)
- **Slider**: Swiper.js
- **Icons**: Lucide React или аналогичные.

#### 4. Стратегия рендеринга

- Большинство страниц генерируются статически (SSG).
- Формы добавления контента (`/add-report`, `/add-event`) могут требовать SSR или клиентских островов (Client Islands) для взаимодействия с API.
