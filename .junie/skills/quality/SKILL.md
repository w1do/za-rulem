### Качество кода, проверка и форматирование

Этот документ определяет стандарты качества, правила форматирования и процессы проверки кода в проекте Campfish.

#### 1. Форматирование кода

Все файлы в проекте должны соответствовать единому стилю для обеспечения читаемости.

- **Инструмент**: Prettier. Конфигурация находится в `.prettierrc`.
- **Правила**:
  - Использовать 2 пробела для отступов.
  - Точка с запятой в конце строк.
  - Одинарные кавычки для строк (кроме JSX/Astro атрибутов).
  - Трейлинг-запятые в объектах и массивах (es5).
- **Именование**:
  - Компоненты (`.astro`, `.tsx`): `PascalCase` (например, `Header.astro`, `UserCard.tsx`).
  - Файлы страниц и контента: `kebab-case` (например, `about-us.astro`).
  - Стили и утилиты: `camelCase`.

#### 2. Проверка кода и Типизация
Проверка кода должна выполняться регулярно в процессе разработки.

- **ESLint**: Используется для проверки логических ошибок и стиля. Конфигурация в `eslint.config.js`.
- **TypeScript**:
  - Обязательное использование типов для всех props в компонентах.
  - Избегать использования `any`. При необходимости использовать `unknown` или детальное описание типа.
  - Использовать интерфейсы для описания структур данных.
- **Astro Check**:
  - Команда `npm run check` проверяет типы в `.astro` файлах.

#### 3. Обработка ошибок

- **Грациозная деградация**: Если данные не загрузились (например, в коллекциях), компонент не должен вызывать "белый экран". Используй проверку на существование данных.
- **Zod Схемы**: Всегда проверяй входящий контент через Zod в `src/content/config.ts`. Это гарантирует, что ошибки контента будут пойманы на этапе сборки.

#### 4. Стандарты Astro и React

- **Client Directives**: Используй `client:*` экономно. Только для интерактивных компонентов (формы, фильтры, модальные окна).
- **Assets**: Используй компонент `<Image />` из `astro:assets` для автоматической оптимизации. Никогда не используй тег `<img>` напрямую для локальных файлов.

#### 5. Чек-лист перед завершением задачи (Submit)

Перед тем как вызвать `submit`, убедись, что:

1. Код отформатирован (`npm run format`).
2. Линтинг прошел без ошибок (`npm run lint`).
3. Типы и структура проверены (`npm run check`).
4. В консоли нет ошибок при локальном запуске (`npm run dev`).
5. Компоненты следуют дизайн-системе из `DESIGN.md`.
