# Навык: source-copy

Этот навык описывает **точный перенос** готовой вёрстки из шаблона `source/*.html` в Astro-страницы и компоненты проекта. Триггер — команда вида «перенеси мне страницу `index-3.html`», «перенеси `services.html`», «собери страницу из шаблона `about.html`».

> Главный принцип: **блок в блок, до каждой мелочи**. Ничего от себя добавлять нельзя — ни стилей, ни блоков, ни новых секций. Разрешено менять **только тексты** (на русский язык) и пути к статике (`images/` → `/images/`). Дизайн, классы, порядок и вложенность разметки остаются идентичными референсу.

---

## 0. Что делает навык

По имени файла из `source/` агент:
1. Находит и **полностью анализирует** исходный HTML.
2. Разбивает его на блоки по HTML-комментариям `<!-- ... Start -->` / `<!-- ... End -->`.
3. Создаёт под каждый блок отдельный `.astro`-компонент в `src/components/{страница}/`.
4. Собирает страницу `src/pages/{slug}.astro`, подключая общие компоненты (`Header`, `Footer`) и созданные блоки в **том же порядке**, что и в референсе.
5. Переводит тексты на русский язык (по правилам проекта), не трогая разметку.

---

## 1. Жёсткие правила (нельзя нарушать)

- ❌ **Не добавлять** новые блоки, секции, обёртки, `div`-ы, классы, атрибуты, инлайн-стили.
- ❌ **Не исправлять** и не «улучшать» вёрстку, классы, отступы, порядок элементов.
- ❌ **Не переписывать** классы на Tailwind. Классы шаблона (`bootstrap`, `custom.css`) переносятся **как есть**.
- ❌ **Не выдумывать** контент вне тематики и вне SEO-запросов.
- ✅ **Можно** менять только видимый текст на русский и `alt`/`title`/`placeholder`.
- ✅ **Можно** править пути к ресурсам: `images/...` → `/images/...`, `css/...`, `js/...` при необходимости.
- ✅ Структура, вложенность и последовательность блоков — **идентичны** оригиналу.

Соответствие проверяется дословно: если удалить тексты, разметка компонентов и разметка референса должны совпадать блок в блок.

---

## 2. Алгоритм переноса

### Шаг 1. Анализ референса
1. Открыть файл `source/{имя}.html` целиком.
2. Определить `slug` страницы (имя файла без `.html`): `about.html` → `about`, `index-3.html` → главную-версию (см. договорённость по именованию ниже).
3. Выделить границы `<head>`, `<body>` и все блоки внутри `<body>` по парам комментариев `<!-- X Start --> ... <!-- X End -->`.

### Шаг 2. Карта блоков
Составить список блоков в порядке следования, например для `index.html`:
- `Preloader` → общий (уже реализован / пропустить, если не используется).
- `Header` → общий компонент `components/shared/header/Header.astro` (не дублировать!).
- `Hero Section` → `components/{страница}/{Страница}Hero.astro`.
- `... Section` → `components/{страница}/{Страница}...astro`.
- `Footer` → общий компонент `components/shared/footer/Footer.astro`.

### Шаг 3. Создание компонентов (блок в блок)
Для каждого содержательного блока создать файл `src/components/{страница}/{Страница}{Блок}.astro`:
- В теле — **точная копия HTML** блока из референса (включая комментарии `Start/End` для наглядности).
- Frontmatter (`---` ... `---`) добавляется пустым или с `Astro.props`, только если блок реально требует данных.
- Тексты переводятся на русский по правилам проекта (см. раздел 4).
- Пути к картинкам приводятся к `/images/...`.

### Шаг 4. Сборка страницы
Создать `src/pages/{slug}.astro`:
- Импортировать `Layout`, общие `Header`/`Footer` и все созданные блоки.
- Разместить компоненты **в точном порядке** референса.
- Заполнить `title`/`description`/`keywords` в `Layout` по SEO-запросу страницы.

### Шаг 5. Формы
Любые формы обратной связи/заявки — **не** переносить как статический HTML. Выносить в отдельный React-компонент-хук (см. `components/forms/*`), отправка на n8n-endpoint (см. `AGENTS.md`). Место формы в разметке совпадает с референсом.

### Шаг 6. Проверка соответствия
1. Сверить порядок и состав блоков страницы с референсом — один в один.
2. Убедиться, что не появилось лишних/пропущенных блоков.
3. Запустить `npm run build` — сборка должна проходить без ошибок.
4. Добавить URL новой страницы в `urls-seo.txt` (см. `AGENTS.md`).

---

## 3. Нейминг и структура (следует навыку `components`)

- Блоки конкретной страницы: `src/components/{страница}/{Страница}{Блок}.astro`.
  - Пример `services.html`: `components/services/ServicesHero.astro`, `components/services/ServicesList.astro`, ...
- Общие блоки (`Header`, `Footer`, `Preloader`): переиспользуются из `components/shared/*`, **не дублируются**.
- Имя компонента — `PascalCase` с префиксом страницы, отражает суть блока.
  - ✅ `ServicesHero.astro`, `AboutTeam.astro`
  - ❌ `Hero.astro`, `Block1.astro`
- Крупные блоки дробить на составляющие, не допуская «портянок» (правило из навыка `components`).

---

## 4. Тексты (правила проекта)

- Все тексты — на русском языке и **от первого лица** (я, меня), без «вы/вас/компания/студия» (см. `AGENTS.md`).
- Наполнение и SEO берутся из папки `seo/` под соответствующий запрос и оформляются по навыку `.junie/skills/seo-write/SKILL.md`.
- Разметку под текст менять нельзя: если в референсе `<h3>...</h3>` — остаётся `<h3>`, меняется только содержимое.

---

## 5. Договорённости по именованию страниц

- `index.html` → `src/pages/index.astro` (главная).
- `about.html` → `src/pages/about.astro`, `services.html` → `src/pages/services/index.astro` и т.п.
- `index-2.html`, `index-3.html` — альтернативные версии главной: перенос выполняется по запросу, `slug` согласуется с задачей (например, отдельная посадочная), но набор блоков переносится из соответствующего референса блок в блок.

---

## 6. Чек-лист завершения

- [ ] Все блоки референса перенесены, лишних нет.
- [ ] Разметка идентична оригиналу (совпадает блок в блок при игнорировании текста).
- [ ] Общие `Header`/`Footer` переиспользованы, не продублированы.
- [ ] Формы вынесены в React-хуки, а не статический HTML.
- [ ] Тексты на русском, от первого лица, по SEO из `seo/`.
- [ ] Пути к статике исправлены на абсолютные (`/images/...`).
- [ ] `npm run build` проходит успешно.
- [ ] URL добавлен в `urls-seo.txt`.
- [ ] Обновлён `SUMMARY.md` по навыку `.junie/skills/documentation/SKILL.md`.
