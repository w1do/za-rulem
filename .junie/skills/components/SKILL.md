### Компоненты и Дизайн-система

Инструкции по созданию и использованию UI-компонентов в проекте на основе [DESIGN.md](../../DESIGN.md).

#### 1. Основные стили (Tailwind)

- **Контейнер**: `max-width: 1280px` (`max-w-[1280px] mx-auto px-4`).
- **Фон сайта**: Светло-серый (`bg-slate-50`).
- **Header**: Прозрачный (`bg-transparent`).
- **Footer**: Серый (`bg-slate-100`).
- **Акценты**: Оранжевый (`bg-orange-500`, `text-orange-500`) для кнопок, иконок и активных элементов.
- **Текст**: `text-slate-900` для заголовков, `text-slate-600` для основного контента.

#### 2. Компоненты Shadcn

Используйте Shadcn для сложных UI-элементов:

- **Dialog (Modal)**: Для открытия подробностей отчетов или событий.
- **Accordion**: Для секции FAQ.
- **Button**: Стилизация через оранжевый цвет (`variant="default"` должен быть переопределен на оранжевый).
- **Badge**: Для категорий (#Рыбалка, #Кемпинг).

#### 3. Паттерн "Карточка" (Base Card)

Все карточки (события, локации, отчеты) должны быть легкими и чистыми, переиспользованы через props, помещены в components и использованы как <ComponentName/>:

```html
<div
  class="bg-white rounded-xl border border-slate-200/60 overflow-hidden transition-all hover:bg-blue-50/30 hover:shadow-sm"
>
  <!-- Изображение -->
  <img src="..." class="w-full h-48 object-cover" alt="..." />
  <div class="p-4">
    <!-- Бейджи -->
    <div class="flex gap-2 mb-2">
      <span class="text-xs bg-slate-100 text-slate-600 px-2 py-1 rounded"
        >...</span
      >
    </div>
    <!-- Контент -->
    <h3 class="text-lg font-semibold text-slate-900">...</h3>
    <p class="text-slate-600 text-sm line-clamp-2">...</p>
    <!-- Кнопка действия -->
    <button
      class="mt-3 text-sm text-orange-500 font-medium hover:text-orange-600 transition-colors"
    >
      Подробнее →
    </button>
  </div>
</div>
```

#### 4. Интеграция Swiper.js

Для слайдеров (например, "Ближайшие события"):

- Инициализация на клиенте (через `<script>` или React/Solid компоненты).
- Настройки: `slidesPerView: 1` для мобильных, `slidesPerView: 3` для десктопа.
- Навигация: Стрелки и пагинация должны быть стилизованы под общий дизайн.

#### 5. Доступность (A11y)

- Использование семантических тегов (`<header>`, `<footer>`, `<main>`, `<section>`).
- Управление фокусом в модальных окнах.
- Достаточный контраст текста.

#### 6. Слои компонентов и нейминг

- Запрещено создавать дубликаты компонентов.
- Перед созданием нового компонента необходимо проверить наличие аналогичного или подходящего по функционалу в проекте.
- Переиспользование существующих компонентов — приоритет.

# Нейминг и структура папок для компонентов страниц

- Компоненты, относящиеся к конкретной странице (например, Home, About), должны быть организованы по слоям.
- Схема размещения: components/{название*страницы}/{название*компонента}.astro
- Пример для страницы Home: components/home/HomeAbout.astro
- Пример для страницы About: components/about/AboutTeam.astro

# Правила именования компонентов

- Название компонента отражает его суть и страницу принадлежности (PascalCase, с префиксом страницы).
- ✅ HomeHero.astro, HomeFeatures.astro, AboutHistory.astro
- ❌ Hero.astro, Features.astro (принадлежность странице неочевидна)
