# Навык: ui-copy

Навык `ui-copy` анализирует **все** CSS-стили проекта в `public/assets/css` и собирает
из них **дизайн-систему** (design tokens + компоненты) в виде набора Markdown-схем.
Цель — иметь точную «карту» UI, чтобы воспроизводить (копировать) интерфейс этого сайта
в новых компонентах/страницах Astro без ручного чтения 11k+ строк CSS.

> Базовый источник правды по стилю — `public/assets/css/main.css`. Остальные файлы
> (`bootstrap.min.css`, `animate.css`, `swiper-bundle.min.css`, `magnific-popup.min.css`,
> `all.min.css`) считаются **вендорными** и в дизайн-систему не выносятся, кроме
> сетки/утилит Bootstrap, которые реально используются в разметке.

## Что входит в анализ (`public/assets/css`)
| Файл | Тип | Роль в дизайн-системе |
|------|-----|------------------------|
| `main.css` | кастомный | **Основной источник** токенов и компонентов |
| `bootstrap.min.css` | вендор | Сетка `container/row/col-*`, утилиты |
| `animate.css` | вендор | WOW-анимации (`fadeInUp` и т.д.) |
| `swiper-bundle.min.css` | вендор | Слайдеры/карусели |
| `magnific-popup.min.css` | вендор | Лайтбоксы/попапы |
| `all.min.css` | вендор | Font Awesome иконки (`fa-*`) |

## Результат работы навыка
Навык формирует папку `design_systems/` (рядом с этим `SKILL.md`):

```
design_systems/
├── SKILL.md          # индекс дизайн-системы, навигация по схемам
├── colors.md         # цветовые токены (:root)
├── typography.md     # шрифты, заголовки h1–h6, текст, section-title
├── sizes.md          # размеры, радиусы, утилиты отступов (pt/pb/mt/mb), брейкпоинты
├── button.md         # .theme-button + style-1..4, состояния, разметка
├── cards.md          # карточки/боксы (pricing-item, why-items и пр.)
├── forms.md          # инпуты, textarea, select, чекбоксы, ajax-response
├── effects.md        # тени, переходы, hover, градиентные бордеры, анимации
└── references/       # готовые примеры использования UI с этими стилями
    ├── buttons.md
    ├── cards.md
    ├── section-title.md
    └── page-example.md
```

## Алгоритм работы навыка
1. **Снять инвентарь стилей.** Прочитать список файлов в `public/assets/css`,
   отделить кастомный `main.css` от вендорных библиотек.
2. **Извлечь токены.** Из блока `:root` в `main.css` собрать все CSS-переменные
   (цвета, радиусы, шрифты, transition, box-shadow) → `colors.md`, `sizes.md`,
   `typography.md`, `effects.md`.
3. **Описать компоненты.** Для каждого ключевого UI-паттерна (`.theme-button`,
   карточки `*-item/*-box`, формы `.form-control`, `.section-title`) задокументировать:
   селекторы, размеры, состояния `:hover/:focus/.active`, модификаторы (`style-1..4`)
   и **обязательно HTML-разметку** для копирования.
4. **Зафиксировать утилиты.** Спейсинг-классы (`.pt-*`, `.pb-*`, `.mt-*`, `.mb-*`,
   шаг 5px) и адаптивные префиксы (`md-*`), сетку Bootstrap → `sizes.md`.
5. **Собрать references.** В `references/` положить реальные сниппеты разметки из
   `src/components/**` (например `Cta.astro`, `Why.astro`, `Pricing.astro`) как
   образцы корректного применения дизайн-системы.
6. **Поддерживать актуальность.** При изменении `main.css` — перегенерировать
   соответствующие схемы (см. правило ниже).

## Правила воспроизведения UI (copy)
- Использовать **только токены** `var(--…)`, не хардкодить цвета/радиусы.
- Кнопки — всегда `<a class="theme-button style-N"><span data-text="…">…</span><i class="fa-…"></i></a>`
  (атрибут `data-text` обязателен — на нём построена 3D-анимация при hover).
- Заголовки секций — обёртка `.section-title` со `<span class="sub-title">` и `<h2>…<span>акцент</span></h2>`.
- Отступы секций задавать утилитами `pt-100 md-pt-80 pb-100 md-pb-80`, а не инлайн-стилями.
- Анимации появления — классы `wow fadeInUp` + `data-wow-delay`.
- Структуру разметки компонента **не менять** — менять только контент и `props`
  (синхронно с навыком `landing-copy`).

## Документирование
Каждая схема в `design_systems/` — самодостаточный Markdown с таблицей токенов/значений
и блоком HTML-примера. Это «прогрессивное раскрытие»: `design_systems/SKILL.md` — точка
входа, остальные файлы открываются по мере необходимости.
