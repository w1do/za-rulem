#### Основные принципы

- **Content-First**: Всегда проверяй `src/content/` перед созданием новых страниц. Если данные могут быть коллекцией, используй `getCollection`.
- **Mandatory Docs**: Документируй все новые слои, этапы разработки и правки. Сжимай информацию в `SUMMARY.md` для понимания процесса LLM.
- **Clean & Safe Code**: Соблюдай правила форматирования и типизации (см. [гайд по качеству](./quality.md)). Запускай проверки перед сабмитом.
- **Type Safety**: Используй TypeScript для всех компонентов. Определяй Zod-схемы для коллекций в `src/content/config.ts`.
- **UI Consistency**: Используй Shadcn компоненты. При создании новых UI-элементов сверяйся с [инструкцией по компонентам](./components/SKILL.md).

#### Как работать с контентом

1. При создании нового элемента контента создавай `.md` файл в соответствующей папке `src/content/`.
2. Используй правильный frontmatter (см. [спецификацию контента](./content/SKILL.md)).
3. Для связи элементов используй соответствующие идентификаторы (например, `locationId`).

#### SEO и Производительность

- Всегда добавляй мета-теги через компонент SEO или в Layout (см. [SEO гайд](./seo/SKILL.md)).
- Используй компонент `<Image />` из `astro:assets`.
- Минифицируй использование клиентского JavaScript. Используй директивы `client:*` только там, где это необходимо.

#### Команды

- `npm run dev` — запуск сервера разработки.
- `npm run build` — сборка проекта.
- `npm run check` — проверка типов и структуры Astro.
- `npm run format` — автоматическое форматирование кода.
- `npm run lint` — проверка кода на соответствие стандартам.
