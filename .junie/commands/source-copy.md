# /source-copy — перенос страницы из шаблона

Перенеси указанную страницу из шаблона `source/*.html` в Astro-страницу и компоненты
проекта. Имя файла берётся из запроса пользователя (например `/source-copy index-3.html`,
`/source-copy services.html`). Если имя не указано — уточни, какую страницу переносить.

Команда полностью наследует правила навыка `.junie/skills/source-copy/SKILL.md` —
перед началом обязательно прочитай его и следуй ему дословно.

## Главный принцип

**Блок в блок, до каждой мелочи.** Ничего от себя вносить нельзя: не менять стили,
не дорабатывать, не добавлять блоки, секции, обёртки, классы или атрибуты. Разрешено
менять **только** видимый текст (на русский язык) и пути к статике (`images/...` → `/images/...`).
Разметка компонентов при удалении текста должна совпадать с референсом один в один.

## Алгоритм (кратко, детали — в навыке source-copy)

1. Открой `source/{имя}.html` целиком и полностью проанализируй разметку.
2. Разбей `<body>` на блоки по парам комментариев `<!-- X Start --> ... <!-- X End -->`.
3. Создай под каждый содержательный блок отдельный компонент
   `src/components/{страница}/{Страница}{Блок}.astro` — точная копия HTML блока.
4. Общие блоки (`Header`, `Footer`, `Preloader`) переиспользуй из `components/shared/*`,
   не дублируй.
5. Собери страницу `src/pages/{slug}.astro`, подключив компоненты **в том же порядке**,
   что и в референсе, и заполни SEO-мету в `Layout`.
6. Формы обратной связи/заявки выноси в React-хуки (`components/forms/*`), отправка на
   n8n-endpoint (см. `AGENTS.md`), а не как статический HTML.

## Правила текста

- Все тексты — на русском и от первого лица (я, меня), без «вы/вас/компания/студия»
  (см. `AGENTS.md`).
- Наполнение и SEO берутся из папки `seo/` под соответствующий запрос и оформляются
  по навыку `.junie/skills/seo-write/SKILL.md`.
- Нейминг и дробление компонентов — по навыку `.junie/skills/components/SKILL.md`.

## Завершение

- Сверь порядок и состав блоков со страницей-референсом — лишних и пропущенных быть не должно.
- Запусти `npm run build` — сборка должна проходить без ошибок.
- Добавь URL новой страницы в `urls-seo.txt` (команда `/pages`).
- Обнови `SUMMARY.md` по навыку `.junie/skills/documentation/SKILL.md`.
