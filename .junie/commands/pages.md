# /pages — список всех страниц сайта

Собери **актуальный** полный список всех URL сайта и выведи его — каждый адрес с новой
строки. Список используется для отправки страниц на переобход/переиндексацию в поисковые
системы, поэтому он должен быть точным и без лишнего.

Скрипта-помощника нет: собирай адреса вручную, напрямую из файлов проекта по алгоритму ниже.

## Домен и формат URL

- Домен и формат берутся из `astro.config.mjs`:
  - `site: 'https://w1do.ru'` — базовый домен.
  - `trailingSlash: 'never'` — **без** завершающего слэша у всех адресов, **кроме** корня.
- Итоговый URL = `https://w1do.ru` + путь.
- Корень пиши как `https://w1do.ru/` (единственный адрес со слэшем на конце).
- Ничего не добавляй, не комментируй и не группируй в финальном выводе — только ссылки,
  по одной на строку. Отсортируй по алфавиту и убери дубли.

## Откуда берутся страницы (источники и алгоритм)

### 1. Статические роуты — `src/pages/**/*.astro`

Пройди по всем `*.astro` в `src/pages`, **пропуская** динамические роуты (имя файла или
папки содержит `[` — например `[cluster]`, `[post]`, `[...slug]`): они строятся из
контента в шагах 2–4.

Правила преобразования файла в путь:
- `index.astro` в папке → путь самой папки (для корня `src/pages/index.astro` → `/`).
- `<name>.astro` → `/<name>` (и с учётом вложенных папок: `<папка>/<name>`).

Текущие статические страницы:
- `index.astro` → `/`
- `about.astro` → `/about`
- `services.astro` → `/services`
- `case.astro` → `/case`
- `pricing.astro` → `/pricing`
- `contact.astro` → `/contact`
- `testimonials.astro` → `/testimonials`
- `blog/index.astro` → `/blog`
- `knowledge/index.astro` → `/knowledge`

### 2. Блог — `src/content/blog/*.md`

Для каждого `*.md` (кроме файлов, начинающихся с `_`): `/blog/<имя-файла-без-.md>`.

### 3. Кейсы — `src/content/case/*.md`

Для каждого `*.md` (кроме `_*`): `/case/<имя-файла-без-.md>`.

### 4. Услуги-кластеры — `src/content/services/<cluster>/`

Каждая подпапка `<cluster>` — это услуга с `hub: true` в `index.md`:
- `<cluster>/index.md` → хаб-страница `/services/<cluster>`.
- Каждый остальной `<cluster>/<post>.md` → посадочная `/services/<cluster>/<post>`.

### 5. Лендинги-кластеры — `src/content/landings/<cluster>/`

- `<cluster>/index.md` → pillar-страница `/<cluster>` (в корне, без префикса `services`).
- Каждый остальной `<cluster>/<post>.md` → spoke `/<cluster>/<post>`.

## Как собирать актуальный список

Не полагайся на память — перечитай реальные файлы проекта на момент запуска:
- статические `.astro` в `src/pages` (шаг 1);
- содержимое `src/content/blog`, `src/content/case` (шаги 2–3);
- подпапки и `.md` внутри `src/content/services` и `src/content/landings` (шаги 4–5).

Затем собери все пути, добавь домен `https://w1do.ru`, отсортируй, удали дубли и выведи
каждый URL с новой строки.

## Правила вывода (строго)

- Только ссылки — по одной на строку, без нумерации, заголовков, пояснений и группировки.
- Все адреса без завершающего слэша, кроме корня `https://w1do.ru/`.
- Отсортировано по алфавиту, без повторов.
